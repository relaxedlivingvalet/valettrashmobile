# Cursor Repo OS

This repo uses a persistent memory system so future sessions can resume work quickly.

## Core idea
- /brain stores the repo memory
- /.cursor/rules stores behavior rules for Cursor
- /cursor-os stores operator docs and setup notes

## Recommended workflow
1. Read the brain
2. Plan the task
3. Implement
4. Update the brain
5. Commit

## Minimum files to read before work
- brain/project_context.md
- brain/architecture.md
- brain/current_state.md
- brain/next_steps.md
