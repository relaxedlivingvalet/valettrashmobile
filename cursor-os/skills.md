# Skills Blueprint

## brain_update
- Purpose: Update all repo brain files after meaningful work.
- When to use: After coding, debugging, refactoring, planning, or documenting.
- Expected output: Summary of what changed and what remains.

## repo_analyze
- Purpose: Analyze the repo and refresh context and architecture docs.
- When to use: At project start, after major refactors, or when context is stale.
- Expected output: Concise repo assessment and updated brain files.

## debug_mode
- Purpose: Investigate issues systematically and find root cause before fixing.
- When to use: Bugs, regressions, broken builds, failing tests, strange behavior.
- Expected output: Root cause, fix, risks, and follow-ups.

## plan_mode
- Purpose: Create implementation plan before coding.
- When to use: Non-trivial work.
- Expected output: Ordered plan with risks and assumptions.

## ship_mode
- Purpose: Wrap up work and prepare repo for handoff.
- When to use: Before commit, review, or pause.
- Expected output: Updated docs plus concise handoff summary.
