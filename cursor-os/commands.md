# Commands Blueprint

## /start
Read all brain files and summarize project purpose, current state, risks, priorities, and best next actions.

## /resume
Continue from brain/current_state.md and brain/next_steps.md. Identify the most logical next task and begin.

## /plan
Create a step-by-step implementation plan with impacted files, risks, assumptions, and recommended sequence.

## /update-brain
Refresh current_state.md, next_steps.md, decisions.md if needed, and append a new entry to change_log.md.

## /analyze
Inspect the repo and refresh project_context.md and architecture.md based on the latest codebase.

## /ship
Prepare the repo for handoff by updating all brain files and summarizing what changed, what remains, and recommended follow-ups.
